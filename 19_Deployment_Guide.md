# 19 — Deployment Guide

## 1. Prerequisites

| Tool | Version | Purpose |
|---|---|---|
| Docker Engine | 24+ | Container runtime |
| Docker Compose | v2 (plugin) | Multi-container orchestration |
| Git | any | Clone the repository |

No other local dependencies are required. Python, Node.js, and PostgreSQL all run inside containers.

---

## 2. Repository Layout (Deployment-Relevant Files)

```
adas-quality-trace/
├── docker-compose.yml          # Production: Nginx + FastAPI + PostgreSQL
├── docker-compose.dev.yml      # Development: PostgreSQL only (FastAPI and Vite run locally)
├── .env.dev                    # Development env vars — committed to git (weak passwords)
├── .env.prod                   # Production env vars  — NOT committed, lives on server only
├── .env.example                # Template listing all required variables
├── nginx/
│   ├── nginx.conf              # Reverse proxy config
│   └── static/                 # Vite build output (generated by `npm run build`)
├── backend/
│   ├── Dockerfile
│   ├── requirements.txt
│   └── main.py
└── frontend/
    └── ...                     # Vite project; build output goes to nginx/static/
```

## 3. Environment Variables

### 3.1 `.env.example` (template — commit this)

```dotenv
# Database
POSTGRES_HOST=db
POSTGRES_PORT=5432
POSTGRES_DB=pqrets
POSTGRES_USER=pqrets
POSTGRES_PASSWORD=

# Backend
BACKEND_HOST=0.0.0.0
BACKEND_PORT=8000
ALLOWED_ORIGINS=http://localhost

# App
APP_VERSION=6.2.0
```

### 3.2 `.env.dev` (committed to git — weak passwords are fine)

```dotenv
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=pqrets
POSTGRES_USER=pqrets
POSTGRES_PASSWORD=pqrets_dev

BACKEND_HOST=0.0.0.0
BACKEND_PORT=8000
ALLOWED_ORIGINS=http://localhost:5173

APP_VERSION=6.2.0-dev
```

### 3.3 `.env.prod` (NOT committed — create on server before first deploy)

```dotenv
POSTGRES_HOST=db
POSTGRES_PORT=5432
POSTGRES_DB=pqrets
POSTGRES_USER=pqrets
POSTGRES_PASSWORD=<strong-random-password>

BACKEND_HOST=0.0.0.0
BACKEND_PORT=8000
ALLOWED_ORIGINS=http://<your-internal-server-ip-or-hostname>

APP_VERSION=6.2.0
```

Create `.env.prod` on the server once:

```bash
cp .env.example .env.prod
# edit .env.prod: set POSTGRES_PASSWORD and ALLOWED_ORIGINS
```

## 4. Development Environment

Start only PostgreSQL in Docker. Run FastAPI and Vite locally for fast iteration with hot-reload.

```bash
# 1. Start PostgreSQL
docker compose -f docker-compose.dev.yml --env-file .env.dev up -d db

# 2. Backend (separate terminal)
cd backend
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
alembic upgrade head               # run DB migrations
uvicorn main:app --reload --port 8000

# 3. Frontend (separate terminal)
cd frontend
npm install
npm run dev
# → http://localhost:5173
# → /api/* proxied to http://localhost:8000 via vite.config.js
```

### Seed demo data (optional, first time only)

```bash
curl -X POST http://localhost:8000/api/v1/admin/seed-demo
```

## 5. Production Deployment

### 5.1 First-time setup

```bash
# 1. Clone the repository
git clone <repo-url> /opt/pqrets
cd /opt/pqrets

# 2. Create .env.prod (edit passwords and ALLOWED_ORIGINS before continuing)
cp .env.example .env.prod
vi .env.prod

# 3. Build frontend static files
cd frontend && npm install && npm run build && cd ..
# output → nginx/static/

# 4. Start all services
docker compose --env-file .env.prod up -d --build

# 5. Seed demo data (optional)
curl -X POST http://localhost/api/v1/admin/seed-demo
```

After step 4, the application is accessible at `http://<server-ip>`.

Migrations run automatically at backend container startup (`alembic upgrade head` is part of the container start command).

### 5.2 Updating to a new version

```bash
cd /opt/pqrets
git pull

# Rebuild frontend if frontend files changed
cd frontend && npm install && npm run build && cd ..

# Restart backend (migrations run automatically on startup)
docker compose --env-file .env.prod up -d --build backend

# If only nginx config changed
docker compose --env-file .env.prod restart nginx
```

## 6. `docker-compose.yml` (Production)

```yaml
services:
  db:
    image: postgres:15-alpine
    restart: always
    env_file: .env.prod
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U $POSTGRES_USER -d $POSTGRES_DB"]
      interval: 10s
      timeout: 5s
      retries: 5

  backend:
    build: ./backend
    restart: always
    env_file: .env.prod
    depends_on:
      db:
        condition: service_healthy
    command: >
      sh -c "alembic upgrade head &&
             uvicorn main:app --host $BACKEND_HOST --port $BACKEND_PORT"

  nginx:
    image: nginx:1.25-alpine
    restart: always
    ports:
      - "80:80"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/conf.d/default.conf:ro
      - ./nginx/static:/usr/share/nginx/html:ro
    depends_on:
      - backend

volumes:
  postgres_data:
```

## 8. `nginx/nginx.conf`

```nginx
server {
    listen 80;
    server_name _;

    root /usr/share/nginx/html;
    index index.html;

    # SPA fallback — non-API routes serve index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Proxy API requests to FastAPI
    location /api/ {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 60s;
    }
}
```

## 9. Useful Commands

| Task | Command |
|---|---|
| View backend logs | `docker compose logs -f backend` |
| View all logs | `docker compose logs -f` |
| Stop all services | `docker compose down` |
| Stop and delete volumes (⚠ destroys DB data) | `docker compose down -v` |
| Open psql shell | `docker compose exec db psql -U pqrets -d pqrets` |
| Check service health | `docker compose ps` |
| Rebuild a single service | `docker compose --env-file .env.prod up -d --build backend` |

## 10. Backup

PostgreSQL data lives in the `postgres_data` Docker volume.

```bash
# Backup
docker compose exec db pg_dump -U pqrets pqrets > backup_$(date +%Y%m%d).sql

# Restore
docker compose exec -T db psql -U pqrets pqrets < backup_20260601.sql
```

Backup frequency and retention policy are defined by the team operating the server.
